# Gemini API対応 手書き答案採点Webアプリ 要件定義書

## 1. 結論

既存の手書き数式ノートWebアプリをベースに、Gemini APIで手書き答案を読み取り・採点するWebアプリへ拡張する。  
画面は「入力ワークスペース」「採点結果・フィードバック欄」の2領域を中心に構成し、入力ワークスペース内で「答案」と「問題・模範解答」を切り替える。  
問題入力欄では、テキスト入力に加えて、iPadの写真ライブラリから問題画像をインポートできるようにする。  
模範解答欄でも、テキスト入力に加えて、画像をインポートできるようにする。  
また、Gemini無料枠の使いすぎを避けるため、残り目安を円グラフ風メーターで表示する。

## 3. 重要な注意

Gemini APIの正確な残り無料枠を、アプリから常に完全取得できる前提にはしない。  
本アプリでは、採点に成功した回数をアプリ側で記録し、設定した1日上限に対する「残り目安」を表示する。

| 項目 | 扱い |
|---|---|
| 真のGoogle側残量 | Google AI Studio等で確認 |
| アプリ内表示 | 採点成功回数ベースの目安 |
| 他端末での利用 | 初期版では反映しない |
| 429エラー | 実際の上限到達の可能性として警告表示 |
| 表示名 | 「残り回数」ではなく「残り目安」とする |

## 4. 目的

iPad上で問題と答案を入力し、Gemini APIを使って手書き答案の読み取り・採点・フィードバック生成を行う。

既存アプリの手書きキャンバス、ペン、円形消しゴム、保存機能、ノート風UIを活かしながら、学習者向けの自動採点アプリとして拡張する。  
紙の問題、教科書、プリント、板書、スクリーンショットなども写真ライブラリから取り込めるようにする。

## 9. Gemini API連携

### 9.1 基本方針

Gemini APIキーはブラウザに直接置かない。  
Vercel上のサーバーレスAPI Routeを経由してGemini APIを呼び出す。

```text
iPad Webアプリ
  ↓
Vercel Serverless Function
  ↓
Gemini API
  ↓
Vercel Serverless Function
  ↓
iPad Webアプリ
```

### 9.2 Gemini API無料枠・レート制限の扱い

Gemini APIの無料枠は、モデル、利用ティア、プロジェクト状態によって変わる。  
公式ドキュメントでは、レート制限は主にRPM、TPM、RPDで評価され、RPDは太平洋時間の午前0時にリセットされる。

| 指標 | 内容 |
|---|---|
| RPM | 1分あたりのリクエスト数 |
| TPM | 1分あたりの入力トークン数 |
| RPD | 1日あたりのリクエスト数 |
| 適用単位 | APIキー単位ではなくプロジェクト単位 |
| リセット | RPDは太平洋時間0:00にリセット |
| 注意 | 指定された制限は保証値ではなく、実際の容量は変動する |

本アプリでは、Google側の正確な残量を直接取得するのではなく、アプリ側で採点成功回数を記録し、残り回数を「目安」として表示する。

### 9.3 APIキー管理

| 項目 | 内容 |
|---|---|
| 保存場所 | Vercel Environment Variables |
| 変数名 | `GEMINI_API_KEY` |
| ブラウザ送信 | しない |
| GitHub保存 | しない |

### 9.4 送信データ

Gemini APIへ送る情報は以下とする。

| データ | 内容 |
|---|---|
| questionText | 手入力された問題文 |
| questionImage | 問題が書かれた画像 |
| modelAnswer | 模範解答 |
| modelAnswerImage | 模範解答が書かれた画像 |
| rubricText | 採点基準 |
| maxScore | 満点 |
| answerImage | 手書き答案画像 |
| outputLanguage | 日本語 |

### 9.5 返却データ

Geminiからの返却はJSON形式に統一する。

```json
{
  "recognizedQuestion": "読み取った問題内容",
  "recognizedAnswer": "読み取った答案内容",
  "recognizedLatex": "読み取った数式のLaTeX",
  "score": 7,
  "maxScore": 10,
  "resultType": "partial",
  "feedback": "考え方は合っていますが、最後の符号に誤りがあります。",
  "mistakes": [
    "2行目で符号が反転しています"
  ],
  "improvements": [
    "途中式をもう少し丁寧に書きましょう"
  ],
  "confidence": "medium"
}
```

## 10. 利用量メーター仕様

### 10.1 基本仕様

| 項目 | 内容 |
|---|---|
| 表示方法 | 円グラフ風メーター |
| 実装候補 | CSS `conic-gradient` または SVG |
| 記録単位 | 初期版は端末単位 |
| 記録場所 | IndexedDB |
| カウント対象 | Gemini APIへの採点成功 |
| 失敗時 | 成功回数には含めない |
| 429発生時 | レート制限警告を表示し、失敗回数に記録 |

### 10.2 計算式

```text
used = successfulRequests
remaining = max(0, dailyLimit - successfulRequests)
usageRate = successfulRequests / dailyLimit
```

### 10.3 表示色

| 状態 | 条件 | 表示 |
|---|---|---|
| 通常 | 使用率80%未満 | 青系 |
| 注意 | 使用率80%以上 | 黄系 |
| 上限付近 | 使用率90%以上 | 赤系 |
| 上限到達 | remaining = 0 | 採点前に警告 |

### 10.4 上限値の設定

`dailyLimit`はアプリ設定として持つ。  
実際のGemini API制限はモデルやプロジェクト状態で変わるため、初期値は固定値にせず、設定画面または環境変数で変更できるようにする。

| 設定 | 内容 |
|---|---|
| 初期値 | 未確定。Google AI Studioの表示値に合わせて設定 |
| 変更方法 | 管理用設定画面またはVercel環境変数 |
| 推奨表示 | 「残り目安」と明示 |

## 11. 採点プロンプト要件

Geminiへ送るプロンプトには以下を含める。

| 項目 | 内容 |
|---|---|
| 役割 | 数学答案の採点者 |
| 入力 | 問題文、問題画像、模範解答、模範解答画像、採点基準、手書き答案画像 |
| 出力 | JSONのみ |
| 採点方針 | 部分点を考慮する |
| 不確実性 | 読み取りが怪しい場合は信頼度を下げる |
| 注意 | 読み取れない文字を推測しすぎない |

プロンプト方針:

```text
あなたは数学答案の採点者です。
問題画像または問題文を読み取り、手書き答案画像を採点してください。
模範解答と採点基準に基づいて、部分点を考慮してください。
出力は必ずJSONのみとしてください。
読み取りが不確実な場合は confidence を low にしてください。
```

## 14. エラー・例外対応

| 状況 | 対応 |
|---|---|
| 問題文・問題画像が両方未入力 | 「問題文または問題画像を入力してください」と表示 |
| 模範解答・模範解答画像が両方未入力 | 「模範解答または模範解答画像を入力してください」と表示 |
| 答案未入力 | 「答案を手書きしてください」と表示 |
| 画像形式非対応 | 「対応していない画像形式です」と表示 |
| 画像サイズが大きすぎる | 圧縮または再選択を促す |
| 画像読み込み失敗 | 「画像を読み込めませんでした」と表示 |
| APIキー未設定 | 「Gemini APIキーが設定されていません」と表示 |
| Gemini API失敗 | 「採点に失敗しました」と表示 |
| Gemini API 429 | 「無料枠またはレート制限に達した可能性があります」と表示 |
| JSON解析失敗 | 生テキストを表示し、再採点を促す |
| 読み取り信頼度low | 人間確認を促す |
| IndexedDB保存失敗 | 「保存に失敗しました」と表示 |
| 利用量取得失敗 | 「使用回数の表示に失敗しました」と表示し、採点自体は継続可能にする |

| 画像送信 | 採点時に問題画像、模範解答画像、答案画像をGeminiへ送信する |
| 注意表示 | 外部AIへ画像を送ることをユーザーに明示する |
| 利用量表示 | 残り目安であり、Google側の保証値ではないことを明示する |

## 18. 実装方針

### 18.1 フロントエンド

| コンポーネント | 役割 |
|---|---|
| QuestionPanel | 問題文、問題画像、模範解答、採点基準を入力 |
| QuestionImagePicker | iPad写真ライブラリから問題画像を選択 |
| ModelAnswerImagePicker | iPad写真ライブラリから模範解答画像を選択 |
| InputTabs | 答案入力と問題・模範解答入力を切り替え |
| AnswerCanvas | 手書き答案、ペン、消しゴムを管理 |
| GradingPanel | 採点結果、フィードバックを表示 |
| UsageMeter | 使用回数、残り目安、円グラフ風メーターを表示 |
| HistoryList | 採点履歴を表示 |
| GradingService | API Routeへ採点依頼を送信 |
| UsageService | 採点成功回数を記録・集計 |

### 18.2 サーバーレスAPI

| API | 内容 |
|---|---|
| `/api/grade` | Gemini APIへ採点依頼を送信 |
| `/api/usage-config` | アプリ側の上限設定を返す |

処理の流れ:

```text
POST /api/grade
  ↓
入力バリデーション
  ↓
Gemini APIキー確認
  ↓
問題文、問題画像、模範解答、模範解答画像、答案画像をGeminiへ送信
  ↓
JSON結果を返却
```

### 18.3 利用量メーター処理

```text
アプリ起動
  ↓
IndexedDBから今日の使用回数を取得
  ↓
dailyLimitと比較
  ↓
円グラフ風メーターを表示
  ↓
採点成功時にsuccessfulRequestsを+1
  ↓
残り目安を更新
```

## 23. 確信度

| 項目 | 確信度 | 理由 |
|---|---|---|
| Gemini APIで画像付き採点アプリを作れる | 高 | 画像入力対応モデルを利用できるため |
| iPad写真ライブラリから画像選択できる | 高 | Safariのファイル入力で画像選択が可能なため |
| 既存手書きUIを流用できる | 高 | Canvas実装を答案欄として使えるため |
| APIキーをVercel環境変数で隠せる | 高 | Serverless Function経由で実装できるため |
| アプリ側で使用回数の目安を表示できる | 高 | 採点成功時にIndexedDBへ記録できるため |
| Google側の真の残量を完全表示できる | 低 | 公式ドキュメントではAI Studioでの確認が案内されており、アプリから正確な残量取得を前提にしにくいため |
| 採点精度が常に安定する | 中 | 筆跡、問題形式、採点基準、問題画像品質に依存するため |
| 無料枠だけで十分運用できる | 低〜中 | 利用回数とGemini API制限に依存するため |
